###几个比较重要的类
Position类：描述一个位置信息，可以进行位置的比较，设置holder，指向位置占用者
Creature类：定义为生物，包含生物的共同特性和方法，如position,CreatureType，可以设定位置。之后的人物都从这里继承。
Container类：拥有若干个数组，数组分别放置葫芦娃势力和敌对势力，数组第一个是leader，每个生物都有一个相对于leader的位置，用于布阵，布阵好之后再上台。
Arena类：一个大擂台，拥有`NxN`的方阵，可以将生物放上去，也可以清空，可以打印当前擂台的情况。
Formation类：布阵器，实现了部分阵型，可以根据给出的Creature数组，leader初始位置，阵型，阵型方向来布阵。

###好处
1.继承机制：所有人物都继承自Creature，并且在Creature中实现了所有人物的共同特性和方法。提高代码抽象度和复用度。
2.接口：葫芦娃实现了Comparable接口，可以在葫芦娃之间做比较。并且使不能比较的不实现该接口。
3.里式替换原则：用Creature指针数组指向不同的子类型，
4.单一职责原则：Contianer用于保存人物及其相对位置，Arena用于将人物放入并展现擂台，Formation只用来布阵
